# 🎓 N-gram Language Model - Demonstration

## ✅ What We Built

A **statistical language model** that learns word patterns from text and generates new text!

### 📊 Model Statistics
- **Training data**: 500 product reviews
- **Vocabulary**: 328 unique words
- **N-grams learned**: 1,195 unique trigrams (3-word sequences)
- **Model type**: Trigram (n=3) - predicts next word based on previous 2 words

## 🧠 How It Works

### Training Phase:
```
Input text: "I love this product"

Step 1: Tokenize
["i", "love", "this", "product"]

Step 2: Build trigrams (n=3)
("START", "i") → "love"
("i", "love") → "this"
("love", "this") → "product"
("this", "product") → "END"

Step 3: Count frequencies
Each context (2 words) stores probabilities for next word
```

### Generation Phase:
```
Start: ("START", "START")
→ Sample next word from learned probabilities
→ Update context with new word
→ Repeat until max_length or "END" token

Example:
Context: ("i", "love") → 60% "this", 40% "the"
Sample: "this"
New context: ("love", "this") → 100% "product"
...
```

## 🔬 Key Concepts Demonstrated

### 1. **Probabilistic Text Generation**
- Model learns P(word | context) from training data
- Generates by sampling from learned distributions

### 2. **Temperature Control**
- **Low (0.5)**: Conservative, picks most likely words
- **High (2.0)**: Creative, explores unlikely words

### 3. **Top-K Sampling**
- Only consider top-K most probable words
- Prevents generating nonsense

### 4. **Local vs Global Coherence**
- ✅ **Local coherence**: Words fit together nicely ("true to size", "very comfortable")
- ❌ **Global coherence**: Contradictions appear ("accurate... too generous")
- This is because n-grams only "see" 2 words back, not full context!

---

## 📈 Limitations (Why We Need Better Models)

1. **Short context window**: Only considers 2 previous words (trigram)
   - Can't understand long-term dependencies
   - No sense of paragraph structure

2. **No semantic understanding**: Purely statistical
   - Doesn't understand what words *mean*
   - Just memorizes which words follow which

3. **Data hungry for rare contexts**: Needs many examples
   - If context never seen in training → stops generating

4. **Fixed vocabulary**: Can't generate new words
   - Limited to 328 words from training data

## 🚀 Next Steps: Evolution to Modern LLMs

### **Level 2: RNN/LSTM** (Next iteration)
- **Better**: Can maintain longer context via hidden states
- **Training**: 10-30 minutes on CPU
- **Output**: More coherent paragraphs

### **Level 3: Transformer** (Modern approach)
- **Better**: Self-attention sees entire input at once
- **Training**: Hours on GPU
- **Output**: Human-like text with global coherence

### **GPT-4 and Beyond**
- **Better**: Trained on trillions of tokens
- **Training**: Months on massive GPU clusters
- **Output**: Indistinguishable from human writing

## 🎯 Educational Value

This N-gram model shows the **fundamental principles** that underlie ALL language models:

1. Learn patterns from data
2. Predict next token probabilistically
3. Sample to generate text
4. Temperature/top-k control creativity

The difference is:
- N-grams: Simple counting
- RNNs: Neural network with memory
- Transformers: Attention mechanism
- GPT-4: Everything above + massive scale
